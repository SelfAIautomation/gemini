# Core Rules

## 実装前

- 既存ファイル（README.md、CLAUDE.md、設定ファイル、既存コード）を確認してから実装を始める。
- 仕様・構成・ファイル名を推測で決めない。不明な場合は確認してから進む。
- 依頼されていない変更を追加しない。追加が必要な場合は理由を提示してユーザー確認を得る。

## 実装中

- 変更は依頼されたスコープの範囲内に留める。
- 新しい依存を追加する場合は、既存の依存リストとの整合性を確認する。
- ファイルを編集する前に必ず `Read` ツールで現在の内容を確認する。
- 設計上の判断（アーキテクチャ変更、DBスキーマ追加等）はユーザーに確認してから進める。

## Push前

- `git diff --name-only` でスコープ外の変更が含まれていないか確認する。
- `.env`、`*.log`、`__pycache__`、`node_modules` などを誤って含めていないか確認する。
- 新しいミスや教訓があれば `docs/ai-lessons/MISTAKE_LOG.md` に追記する。

## 訂正を受けたとき

- 訂正内容を要約し、根本原因を特定する。
- `docs/ai-lessons/MISTAKE_LOG.md` に記録する。
- 再発可能性が高い場合は `.claude/rules/10-known-mistakes.md` にルールとして追加する。
- Push前チェックで検出できる場合は `.claude/rules/20-pre-push-checklist.md` に追加する。
